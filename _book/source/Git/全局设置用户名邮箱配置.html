全局设置用户名邮箱配置

  • 1. [git命令：全局设置用户名邮箱配置]
  • 2. git生成并添加SSH key
    • 2.1. ④ ssh-keygen -t rsa -C "1234@qq.com"
    • 2.2. HTTPS提交报错如下: (身份验证错误)
  • 3. 每次都提示输密码解决办法就是加上本地凭证

1. [git命令：全局设置用户名邮箱配置]

1、查看git配置信息

git config --list

2、查看git用户名

git config user.name

3、查看邮箱配置

git config user.email

4、全局配置用户名

git config --global user.name "nameVal"

5、全局配置邮箱

git config --global user.email "eamil@qq.com"

2. git生成并添加SSH key

① cd ~/.ssh/ 【如果没有对应的文件夹，则执行 mkdir ./.ssh】

② git config --global user.name "xb12369"

③ git config --global user.email "1234@qq.com"

2.1. ④ ssh-keygen -t rsa -C "1234@qq.com"

4、找到C:\Users\xb12369.ssh 目录，里面有两个文件：id_rsa和id_rsa.pub

5、配置ssh【这里是id_rsa.pub里面的内容啊】

2.2. HTTPS提交报错如下: (身份验证错误)

fatal: Authentication failed for “https://********”

解决办法:

git config --system --unset credential.helper

执行此命令会在git push时候输入用户名和密码,如果输入正确就能提交代码

3. 每次都提示输密码解决办法就是加上本地凭证

git config --global credential.helper store

再次输入用户名密码后不再提示
